# 人口数据库管理系统调研与适配建议

## 1. 已有项目文档结论

根据 `doc` 文件夹中的需求规约、数据库设计、项目计划书和数据库总表，本项目定位为一个面向人口信息管理的人口数据库管理系统，核心目标是完成“户-人-证-迁移记录”的一体化数据管理。

项目已有文档中明确的核心模块包括：

- 登录与权限管理：用户登录、修改密码、会话管理、角色权限、登录日志。
- 家庭户口管理：立户、户口成员维护、分户、并户、注销、查询打印。
- 常驻人口管理：人口登记、信息修改、注销、查询统计。
- 户口迁入迁出：迁入申请与办理、迁出办理、迁移进度查询。
- 流动人口管理：流动人口登记、居住证办理、离宿登记、统计分析。
- 重点人口管理：重点人口建档、跟踪、撤控。
- 证件管理：身份证、户口簿、居住证等证件信息维护。
- 统计与日志：综合统计看板、报表导出、操作日志、数据备份。

项目计划书中的优先级划分适合继续沿用：

- P0：登录、户口管理、常驻人口管理、迁入迁出管理、证件管理、数据库基础表结构。
- P1：流动人口管理、重点人口管理、多条件查询、基础统计。
- P2：统计看板、操作日志、数据导出、证件有效期提醒、软删除。

## 2. 外部调研结论

### 2.1 人口登记/户籍类系统思路

联合国统计司关于 population registers 的说明指出，人口登记系统通常围绕单个人口记录维护出生日期、性别、到达/离开日期等基础信息，并可与其他登记数据进行关联。[UNSD Population Registers](https://unstats.un.org/unsd/demographic-social/sconcerns/population_registers/)

CRVS/人口登记资料也强调，人口登记具有行政管理功能，记录生命事件和家庭关系等信息。[Official Statistics Handbook](https://projects.officialstatistics.org/hb-mgnt-org-nss/handbook/chapters/C12/12_7_Household_master_sample%2C_civil_registration_and_vital_statistics_and_population_registers.html)

《The Household Registration System》相关论文介绍了面向家庭纵向调查的人口登记软件系统，核心思想是以 household 和 person 为基础，维护家庭、成员、迁移、事件等关系数据。[Demographic Research PDF](https://www.demographic-research.org/volumes/vol2/6/2-6.pdf)

这些资料和本项目的“户-人-证-迁移记录”模型基本一致，说明当前数据库设计方向合理。

### 2.2 Spring Boot + MyBatis 工程实践

MyBatis 官方 Spring Boot Starter 说明中，3.0.x 版本适配 MyBatis 3.5、MyBatis-Spring 3.0、Java 17+ 和 Spring Boot 3.2-3.5，符合本项目当前 Spring Boot 3.5.3 + JDK 17 技术栈。[mybatis-spring-boot-starter](https://github.com/mybatis/spring-boot-starter)

MyBatis 官方测试文档建议 Mapper 测试可由 Mapper 接口、Mapper XML、数据库脚本和 JUnit 测试组成，这与老师要求的“mapper 两个文件，单元测试一个文件”一致。[MyBatis Unit Test Wiki](https://github.com/mybatis/mybatis-3/wiki/unit-test)

MyBatis Spring Boot Test 文档提供 `@MybatisTest` 这类纯 Mapper 测试方式，后续如果只测试 Mapper 层，可以考虑从 `@SpringBootTest` 进一步收敛到 Mapper 切片测试。[MyBatis Spring Boot Test](https://mybatis.org/spring-boot-starter/mybatis-spring-boot-test-autoconfigure/)

开源项目 RealWorld Spring Boot + MyBatis 展示了 CRUD、认证、分页等常见后端能力，可以作为接口分层和分页风格的参考。[spring-boot-realworld-example-app](https://github.com/gothinkster/spring-boot-realworld-example-app)

## 3. 对当前项目的适配建议

### 3.1 当前已适配内容

当前项目已经具备以下基础：

- Spring Boot 3.5.3 + JDK 17。
- MyBatis Mapper 接口 + XML 映射文件。
- MySQL 数据库 `population_miniterm`。
- 数据库脚本 `doc/database/population_miniterm.sql`。
- Log4j2 日志配置。
- 居民信息 `residents` 的增删改查接口。
- `ResidentMapperTests` 已覆盖新增、查询、修改、删除。

这部分可以作为老师作业和后续模块开发的基础样板。

### 3.2 建议优先落地的模块顺序

建议不要一次性铺开所有表的接口，先按 P0 完成可演示闭环：

1. 人口管理：`person`
   - 新增人口、修改人口、删除/注销人口、按身份证号查询、分页查询。
   - 可复用当前 `ResidentMapper` 的接口/XML/测试结构。

2. 家庭户口管理：`household`、`household_member`
   - 新建立户、添加成员、查询户口成员。
   - 重点验证 household 与 person 的外键关系。

3. 迁入迁出管理：`migration_in`、`migration_out`
   - 新增迁入/迁出记录。
   - 同步更新 `person.status`。
   - 需要加事务，避免只写迁移记录但人口状态没更新。

4. 证件管理：`certificate`
   - 新增证件、修改证件状态、按证件号查询。
   - 可增加证件号唯一校验和有效期校验。

5. 操作日志：`operation_log`
   - 记录新增、修改、删除等关键操作。
   - 答辩时可以展示“操作可追溯”亮点。

### 3.3 建议统一代码结构

每个业务模块建议保持同样结构：

```text
module/
  Xxx.java
  XxxRequest.java
  XxxMapper.java
  XxxService.java
  XxxController.java

resources/mapper/
  XxxMapper.xml

test/java/.../module/
  XxxMapperTests.java
```

当前 `ResidentMapper.java`、`ResidentMapper.xml`、`ResidentMapperTests.java` 可以作为模板。

### 3.4 测试适配建议

每个核心 Mapper 至少覆盖：

- insert：新增后查数据库记录数或主键。
- select：按主键或唯一字段查询，断言字段值一致。
- update：修改字段后重新查询，断言修改成功。
- delete：删除后查询为空。

涉及业务状态变化的模块，例如迁入迁出，应额外测试事务行为：

- 新增迁入记录成功后，`person.status` 应同步变更。
- 迁出记录失败时，人口状态不应被错误修改。

### 3.5 答辩展示建议

答辩时建议围绕三条主线讲：

1. 数据模型：说明系统不是单表堆字段，而是按“户-人-证-迁移记录”建模。
2. 业务闭环：演示从新增人口、建立户口、迁入迁出、证件维护到查询统计的流程。
3. 工程规范：展示 MyBatis XML、单元测试、Log4j2 日志、数据库脚本和 Git 提交记录。
