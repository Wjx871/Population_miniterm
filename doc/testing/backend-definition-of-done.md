# 后端 Definition of Done

模块只有全部通过本清单才能标记 `COMPLETE`；存在类或表不等于完成。

## 1. 功能与数据

- [ ] V1 范围、输入、输出、状态和值域已冻结，无重复入口或第二套模型。
- [ ] MySQL 8 主结构存在，主键、外键、唯一约束、检查约束和高频索引合理。
- [ ] 从规定基线执行增量脚本成功；同一脚本连续执行两次不破坏数据且结果可解释。
- [ ] H2 `schema.sql` 覆盖该模块全部生产表、关键列和足以发现错误的约束。
- [ ] Entity/领域对象、Request/Response DTO 完整；不直接泄露不应暴露的持久化字段。
- [ ] Controller 提供冻结范围内完整 API；HTTP 方法、状态码和 `ApiResponse<T>` 一致。
- [ ] Service 实现状态、幂等、并发版本、事务和跨表一致性规则。
- [ ] Mapper 接口与 XML 成对存在，字段映射、分页、排序、锁与更新行数判断正确。

## 2. 安全与审计

- [ ] 除明确匿名端点外，每个 API 都有最小权限 `@PreAuthorize`。
- [ ] 列表、详情、导出、下载和写入均执行 `DataScopeCriteria` 或等价且可证明的范围判断。
- [ ] 身份证号、手机号、详细地址、证件号等默认脱敏；查看完整值使用独立权限。
- [ ] 新增、修改、状态变更、执行、导出、下载等重大操作写入操作日志；失败访问按规则记录。
- [ ] 日志、异常和文件名不包含密码、Token 或未脱敏敏感值。
- [ ] 核心人口与家庭户不物理删除；历史关系和业务记录可追溯。

## 3. 校验与异常

- [ ] Bean Validation 覆盖必填、长度、格式和值域；服务层覆盖跨字段与跨表规则。
- [ ] 身份证使用校验位和日期算法，不以 18 位正则代替；生日/性别输入与解析结果一致。
- [ ] 重复提交、非法状态迁移、越权、版本冲突、记录不存在均有稳定错误与清晰消息。
- [ ] 审批仅单级；批准不自动执行专业业务，必须走显式 execute。

## 4. 测试与文档

- [ ] 正常、边界、校验失败、未认证、无权限、越范围、并发冲突、事务回滚均有自动化测试。
- [ ] H2 全量 `mvnw clean test` 通过，无删除、禁用或跳过既有测试。
- [ ] 真实 MySQL 8 完成初始化、升级、CRUD/流程、约束、回滚及重复迁移验证，并记录版本与命令。
- [ ] API 文档含路由、权限码、数据范围、脱敏、请求/响应示例、状态值和错误。
- [ ] README/部署/演示文档与真实代码一致。
- [ ] 无废弃 Controller、重复 API、双人口模型或孤立表。
- [ ] `git diff --check` 通过；工作区只包含本阶段授权文件；阶段可独立回归和回滚。

## 5. 审核记录模板

| 项目 | 结果 | 证据 |
|---|---|---|
| Maven/H2 | PASS/FAIL | 测试数、失败/错误/跳过、耗时 |
| MySQL 初始化 | PASS/FAIL | MySQL 版本、脚本、日志/报告 |
| MySQL 增量与重复执行 | PASS/FAIL | 起始版本、执行两次结果 |
| 权限/范围/脱敏/审计 | PASS/FAIL | 测试类和权限矩阵 |
| API/部署文档 | PASS/FAIL | 文件路径 |
| diff/status | PASS/FAIL | 命令结果 |

任何一项 FAIL 或“未验证”时，状态最多为 `PARTIAL`；代码无法正常运行时标记 `BROKEN`；存在两套有效实现时标记 `DUPLICATE`。
