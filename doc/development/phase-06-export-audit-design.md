# 第六阶段导出与审计设计

普通导出用于日常工作，只允许白名单字段、当前数据范围和强制脱敏，直接生成文件。敏感导出可包含完整身份、手机或地址，因此必须通过专业申请、材料、审批和显式执行；审批通过本身不产生文件。

客户端不能提交 SQL、表名、排序表达式或文件路径。`ExportPolicy` 维护模块、过滤器和字段白名单，`ExportMapper.xml` 通过固定 `<choose>` 查询实现普通 MyBatis 数据访问，没有字符串拼接任意 SQL。

`SensitiveDataMaskingService` 集中处理身份证、手机、居住证号、地址、文件名和审计 detail。页面完整查看由 `sensitive-data:view-full` 控制；普通导出即使操作者有完整查看权限，也强制使用 always-mask 方法。

XLSX 使用最小 POI OOXML 依赖。号码写为文本，公式触发字符转义，工作表名和文件名由服务端生成。临时文件与最终文件限制在规范化的 `EXPORT_DIR` 下，失败会清理文件，响应不暴露路径。

`data_export_request` 是敏感导出的专业审批详情，保存不可执行的受控条件快照和批准额度；`data_export_log` 记录每一次真实生成、文件摘要、下载和清理状态。`operation_log` 记录系统行为，`sys_approval_log` 记录审批与执行轨迹，居住证日志继续只记录证件生命周期。

生成使用唯一导出编号、专业申请唯一约束、状态条件更新和版本控制。重复执行不能生成第二份完成文件。清理扫描只选 COMPLETED 过期记录，每条在独立事务处理；已 EXPIRED 的记录不会重复处理。

本阶段不实现任意 SQL、大数据流式/异步平台、消息队列、云存储或多级审批。第七阶段应进行真实 MySQL 8 迁移、端到端权限验收、文件部署权限检查和最终集成验收。
