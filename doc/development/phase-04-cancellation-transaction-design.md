# 第四阶段注销事务设计

审批只代表允许办理，不能自动执行高风险注销。`cancellation_record` 保存专业字段与执行快照，通用申请继续负责审批状态和轨迹。

`person`、`household` 都是永久基础档案，注销只改变状态；不允许物理删除。人员注销时，`residence_archive` 保存个人当前户籍快照；家庭户销户时，`household_archive` 保存户号、原户主、地址、区划、户类型和建立日期，两者职责不同。

人员注销事务依次锁定申请、专业记录、人员、户籍与家庭户，写个人户籍归档，删除当前户籍，成员置为 `CANCELLED`，处理新户主或待销户状态，人员置为 `DECEASED/CANCELLED`，最后同步完成专业记录、申请和审计日志。家庭户销户事务重新检查无有效成员、无当前户籍、无待迁入，写家庭户归档，将主档置为 `CANCELLED`，再同步完成状态与日志。

唯一申请关联、乐观锁、条件更新、行锁和唯一归档约束保证幂等。可替换事务钩子覆盖归档后、删除户籍后、成员更新后、人员更新后、家庭户归档后及家庭户状态更新后异常，生产接口不暴露故障参数。

本阶段不支持注销撤销、恢复户籍或真正家庭户合并。后续流动人口、居住证模块只能读取已注销状态，不得隐式恢复。

当前无真实 MySQL 环境，V4_004 仅完成静态检查并由 H2 自动化验证业务；MySQL 8 的旧数据兼容、CHECK 行为与重复执行仍待实机验证。
