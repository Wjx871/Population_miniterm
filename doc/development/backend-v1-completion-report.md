# Backend V1.0.0 完成报告

本项目定位为“课程设计级、可完整演示和部署的后端 V1”，不是生产级政务系统。

已完成单一 person 人口主档、家庭户与成员、户籍迁移、注销、流动人口与居住证、行政区划、数据字典、通用证件、重点人口、申请材料审批、显式执行、导出、综合查询、统计、日志、权限、范围、脱敏、审计和可选 Redis。架构冻结为 Java 17、Spring Boot 3.5.3、普通 MyBatis、Spring Security/JWT、MySQL 8；Redis 只缓存参考/聚合数据和注销 jti。

Backend V1 使用 V4_001—V4_010，最终自动化测试 321 项；MySQL 8.4.10 全新安装和升级链、Redis 7 三模式以及可执行 JAR 均通过。原生部署不要求 Docker。

明确排除：多级审批、微服务、消息队列、政务联网、实体制卡、短信邮件、AI 风险预测、全国完整区划，以及用户/角色/部门在线管理 CRUD。主要风险是课程演示数据不可用于真实业务，JWT 密钥轮换会使旧 Token 失效，文件与数据库备份需由部署方统一治理。

后续前端联调应以 `doc/api/backend-api-index.md` 为准，重点核对五角色菜单、403/404 边界、分页模型、默认脱敏、专业申请与 execute 分离、Redis 健康状态；不得恢复 `/api/residents` 或绕过专业入口。
