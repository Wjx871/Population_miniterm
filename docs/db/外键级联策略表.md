# 外键级联策略表 v4.0

> 设计原则：**默认 `RESTRICT`**（即不允许删除被引用行）。少数场景用 `SET NULL`（用于审计字段）。
> 全表清单按职责（户籍、迁移、注销）筛选；引用他人表的外键同等对待。

## 1. 居民侧

| 表 | 外键 | 级联策略 | 理由 |
|----|------|---------|------|
| `person` | 无外键（顶级主档） | - | - |
| `household` | `fk_household_head (head_person_id) → person` | RESTRICT | 户主人口不能直接 DELETE；变更走 `changeHead` |
| `household` | `fk_household_region (region_code) → admin_region` | RESTRICT | 区划被户引用，不允许删区划 |
| `household` | `fk_household_department (department_id) → sys_department` | RESTRICT | 部门长期引用 |
| `household_member` | `fk_member_household (household_id) → household` | RESTRICT | 户未销户前不能删 |
| `household_member` | `fk_member_person (person_id) → person` | RESTRICT | 人口未彻底注销前不能删 |
| `household_member` | `fk_member_application (source_application_id) → business_application` | RESTRICT | 申请单不能删 |
| `residence_registration` | `fk_registration_person (person_id) → person` | RESTRICT | 与 person 同步注销/迁移走 archive |
| `residence_registration` | `fk_registration_household (household_id) → household` | RESTRICT | 户销户先处置成员 |
| `residence_registration` | `fk_registration_region (region_code) → admin_region` | RESTRICT | 同上 |
| `residence_archive` | `fk_archive_person` | RESTRICT | 历史快照不能丢 |
| `residence_archive` | `fk_archive_household` | RESTRICT | 历史快照不能丢 |
| `residence_archive` | `fk_archive_operator (archive_operator_id) → sys_user` | RESTRICT | 操作人记录不能丢 |
| `migration_in` | `fk_migration_in_application` | RESTRICT | 申请单长期引用 |
| `migration_in` | `fk_migration_in_person` | RESTRICT | 人口不能删 |
| `migration_in` | `fk_migration_in_from_region` | RESTRICT | 区划 |
| `migration_in` | `fk_migration_in_to_household` | RESTRICT | 目标户不能删 |
| `migration_in` | `fk_migration_in_to_region` | RESTRICT | 区划 |
| `migration_in` | `fk_migration_in_new_reg (new_registration_id) → residence_registration` | **SET NULL** | 迁入办结后回填；如新登记被回滚置 NULL 也合理 |
| `migration_in` | `fk_migration_in_operator` | RESTRICT | 操作人 |
| `migration_out` | `fk_migration_out_archive (archive_id) → residence_archive` | RESTRICT | 必须保留归档 |
| 其他 migration_out 外键 | 全部 RESTRICT | 与 in 一致 |
| `cancellation_record` | `fk_cancel_application` | RESTRICT | 申请单 |
| `cancellation_record` | `fk_cancel_person` | RESTRICT | 人口 |
| `cancellation_record` | `fk_cancel_household` | RESTRICT | 户 |
| `cancellation_record` | `fk_cancel_archive (archive_id) → residence_archive` | RESTRICT | 必须保留 |
| `cancellation_record` | `fk_cancel_operator` | RESTRICT | 操作人 |

## 2. 引用 `sys_user`、`sys_department`、`admin_region` 等基础表的字段

所有引用统一 RESTRICT；理由：
- 这些是系统基础档案，**不应**因业务数据删除；
- 启用/停用走 `enabled_flag`/`status` 字段，不通过 DELETE。
- 删除区划会破坏所有户籍引用，强制走"先迁移所有户再删区划"的运维流程。

## 3. 修改级联策略的边界

任何修改 `ON DELETE` / `ON UPDATE` 的 PR 必须：
1. 列出被影响的下游业务流；
2. 调整对应归档策略；
3. 同步 `sql/sanity_check_*.sql` 与本表。
