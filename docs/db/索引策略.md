# 索引策略 v4.0

> 设计原则：
> 1. 高频查询必须命中索引；
> 2. 复合索引最左前缀用尽；
> 3. 索引服务于业务场景而非炫技；
> 4. 不重复建索引（同等列不重复）。

## 2. 索引使用原则

- 不在 SELECT 频繁但过滤性差的列加索引（如 `gender_code` 50/50 分布）。
- 字符串字段使用前缀长度 `idx_person_name (name(20))`（如超长）可减半索引体积。
- 经常做 `GROUP BY` 的列不需要额外索引（与现有索引匹配即可）。
- 计数报表走专用视图（`v_region_population_summary`）。

## 3. 索引演进策略

任何新增索引必须：
1. 给出预计查询与最左前缀用法；
2. 删除可被合并的冗余索引；
3. 在 `indexes_strategy.md` 中追加一节说明。
