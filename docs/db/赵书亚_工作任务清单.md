# 赵书亚：数据库设计与户籍核心业务 — 工作任务清单（v4.0）

> 角色定位：**数据库负责人 + 核心户籍业务后端负责人**
> 数据核心：9 张负责表 `region / data_dictionary / person / household / household_member / residence_registration / residence_archive / migration_in / migration_out / cancellation_record`
> 关键边界：**当前户籍 (residence_registration) 与历史归档 (residence_archive) 必须物理分离，迁出/注销必须在同一事务内完成「生成历史快照 → 移除当前登记 → 更新相关状态」。**

## 6. Service 层（**这一层是核心**）

### 6.1 通用方法（在 `ResidenceRegistrationServiceImpl` 上提）

```text
@Transactional
void archiveAndRemove(Long personId, String archiveTypeCode, String archiveReasonCode,
                     Long sourceApplicationId, Long operatorId, LocalDate archiveDate) {
    1) SELECT FOR UPDATE 旧 registration（可能没有 → 空操作）
    2) INSERT residence_archive（拷贝快照字段；end_date_snapshot = archiveDate）
    3) DELETE 旧 registration
    4) UPDATE person.record_status_code = 'CANCELLED'  （仅当 archiveType=PERSON_CANCEL）
    5) UPDATE household_member.member_status='LEFT'|'CANCELLED'，leave_date = archiveDate
}
```

| 文件 | 必做事项 |

### 6.3 事务可参考写法（伪代码）

```java
@Transactional(rollbackFor = Exception.class)
public void completeOut(Long outId, Long operatorId) {
    MigrationOut out = migrationOutMapper.findByApplicationIdForUpdate(outId)
            .orElseThrow(() -> new NotFoundException("迁出记录不存在"));
    if (out.getCompletedAt() != null) {
        throw new BizException("该迁出记录已办结，不可重复提交");
    }
    // 1. 取旧当前登记
    ResidenceRegistration old = residenceRegistrationMapper
            .findByPersonForUpdate(out.getPersonId())
            .orElse(null);

    // 2. 生成归档快照（事务边界内执行）
    if (old != null) {
        ResidenceArchive snapshot = SnapshotCopier.fromRegistration(
                old, "MIGRATION_OUT",
                out.getReasonCode(),
                out.getOutDate(),
                operatorId,
                out.getApplicationId());
        residenceArchiveMapper.insert(snapshot);
        out.setArchiveId(snapshot.getArchiveId());

        // 3. 删除当前登记
        residenceRegistrationMapper.deleteByPersonAndId(
                old.getPersonId(), old.getRegistrationId());

        // 4. 家庭成员关系置 LEFT
        householdMemberMapper.updateStatusBatch(
                List.of(out.getPersonId()), "LEFT", out.getOutDate());
    }

    // 5. 写回迁出记录
    out.setOperatorId(operatorId);
    out.setCompletedAt(LocalDateTime.now());
    migrationOutMapper.updateById(out);
}
```

## 12. 推进顺序建议（推荐）

1. **Phase 1 — DB & 文档（最先建立基准）**
   - `schema.sql` 补 CHECK / UNIQUE / 索引
   - `init.sql` 字典复核
   - 写出 `docs/db/E-R图_v4.0.md` / `数据字典_v4.0.md` / `核心设计说明_户籍与归档.md`

   - `BizException` 等 8 个异常类 + `GlobalExceptionHandler`
   - `IdCardValidator` / `PhoneValidator` / `RegionUtil` / `SnapshotCopier` / `IdentityMasker`
3. **Phase 3 — DTO**
   - 9 个入参 DTO，配齐 Validation 注解
4. **Phase 4 — Mapper 关键 SQL**
   - 在 8 个职责内 Mapper 补 `findBy*ForUpdate` / 批量更新
5. **Phase 5 — Service 事务核心**
   - 公共方法 `archiveAndRemove`
   - `MigrationOutServiceImpl.complete` ← **头号战场**
   - `CancellationRecordServiceImpl.completePersonCancellation`
   - `ResidenceRegistrationServiceImpl.register`

   - `CancellationRecordServiceImpl.completeHouseholdCancellation`
   - `HouseholdServiceImpl` 立户 / 换户主 / 销户
   - `HouseholdMemberServiceImpl` 转户
6. **Phase 6 — Controller 新端点**
   - 户籍 / 迁入 / 迁出 / 注销的 detail、complete、precheck、by-batch
7. **Phase 7 — 测试数据 & 一致性 SQL**
   - `data.sql` 扩到 12 人以上 + `migration_*` + `cancellation_*`
   - `test_integrity_asserts.sql` / `test_archive_consistency.sql`
