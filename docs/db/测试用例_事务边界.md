# 测试用例：事务边界

> 以下剧本对应 Spring 服务层的事务回放，配合 SQL 一致性探针执行。
> 每步"期望"列对应 sanity_check 或 test_archive_consistency 输出的 PASS。

## 剧本 A：建户 → 立户主 → 添加成员 → 迁入（外来）→ 注销

1. `POST /api/persons` 新增人员 A（外来，ACTIVE）。
2. `POST /api/households/establish` 立户 H_a，户主 = A，address / region_code 必填。
3. `POST /api/residence-registrations` 登记 A 当前户籍到 H_a（一人一条；事务内 FOR UPDATE）。
   - 期望：`uk_registration_person` 立即生效，二次登记同 person 被拒。
4. `POST /api/cancellation-records` 创建 A 的死亡注销记录。
5. `GET /api/cancellation-records/precheck-person/{A}` 返回 passable=true。
6. `PUT /api/cancellation-records/{cancelId}/complete-person?operatorId=...`
   - 事务内：
     - INSERT residence_archive（PERSON_CANCEL）
     - DELETE residence_registration
     - UPDATE household_member（CANCELLED）
     - UPDATE person（CANCELLED）
     - UPDATE cancellation_record
   - 期望：迁移出与销户后，sanity_check_one_per_person / sanity_check_household_head / test_archive_consistency 全部 PASS。

## 剧本 B：同市跨区联办（批次号 BATCH2026001）

1. A 在 H_a（110101）中已是 CURRENT。
2. 创建迁出记录 `POST /api/migration-out`（CROSS_DISTRICT，transfer_batch_no=BATCH2026001）。
3. 创建迁入记录 `POST /api/migration-in`（CROSS_DISTRICT，target=H_b 110102）。
4. **办结顺序由业务流控制**：先办迁出，再办迁入。
   - `PUT /api/migration-out/{outId}/complete?operatorId=...`
     - 事务：归档 + 删登记 + LEFT 成员。
   - `PUT /api/migration-in/{inId}/complete?operatorId=...`
     - 事务：发现 person 已无当前登记（已被迁出归档），INSERT 新登记；INSERT household_member CURRENT 行。
5. 期望：v_migration_batch_detail 中 in_count=1, out_count=1, batch_status='BOTH'。

## 剧本 C：销户前置校验

1. H_x 中仍有 N 名 CURRENT 成员。
2. 户主调用销户 `PUT /api/households/{id}/disable`。
3. 期望：抛 `HouseholdNotEmptyException`（409）。
4. 通过数据修复（迁出/注销所有 CURRENT）后，再次销户成功。

## 剧本 D：并发一人一条

- 两个终端同时给同 person 注册当前户籍：
  - 先者：事务内 FOR UPDATE 锁空行，INSERT 成功。
  - 后者：FOR UPDATE 锁已被先者持有 → 等待 → COMMIT 后看到 existing != null → 抛 `PersonAlreadyHasRegistrationException`。
- 期望：service 抛 BizException 由 GlobalExceptionHandler 翻译为 `Result.error(409, ...)`。

## 剧本 E：迁移归户过程中的故障

- 迁出办结中途 INSERT residence_archive 失败（如 FK 缺失/CHECK 失败）：
  - 事务回滚，无 archive、无 registration DELETE、无 member LEFT 更新。
  - 期望：DB 中所有表的存量保持原状。

## 跑完所有剧本后跑一致性 SQL

```bash
mysql -u root -p population_miniterm \
  < sql/sanity_check_one_per_person.sql
mysql -u root -p population_miniterm \
  < sql/sanity_check_archive_link.sql
mysql -u root -p population_miniterm \
  < sql/sanity_check_household_head.sql
mysql -u root -p population_miniterm \
  < sql/test_archive_consistency.sql
mysql -u root -p population_miniterm \
  < sql/test_one_per_person.sql
mysql -u root -p population_miniterm \
  < sql/test_household_head_consistency.sql
```

全部 PASS 即事务边界正确。
