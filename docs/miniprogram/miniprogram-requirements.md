# 微信小程序移动管理端 V1 范围

## 产品定位

本项目是“人口数据库管理系统内部移动管理端”，供已有系统账号在移动场景下查询和审批。它不是公众居民办事端，不提供微信 OpenID 登录、公众注册、居民实名认证或政务平台联网。

## V1 功能

- 系统账号登录、`/api/auth/me` 会话恢复和服务端退出。
- 权限驱动的移动工作台，展示真实统计和可用入口。
- 人口列表、分页查询和脱敏详情，只读。
- 家庭户列表、分页查询、详情与成员关系，只读。
- 当前账号数据范围内的申请列表、申请详情、材料状态、审批轨迹和最终执行状态。
- 审批人员的待审批、已审批、材料查看、通过和驳回。
- 个人中心、权限摘要、数据范围、后端健康状态和退出。

## 核心约束

- 复用现有 Spring Boot `/api`、JWT、`ApiResponse<T>`、五角色权限、数据范围与服务端脱敏结果。
- 页面入口、操作按钮和发起请求均以 `/api/auth/me` 返回的 `permissions` 为唯一授权依据，不根据角色名推断权限。
- 审批通过不代表专业业务已经执行。迁移、注销、重点人口、流动人口、居住证及敏感导出仍需专业 `execute` 权限和显式执行，本移动端 V1 不提供这些执行操作。
- 本地不长期缓存人口、家庭户、申请、审批、材料或下载文件；只保存 JWT、用户基本信息、权限和“记住账号”的用户名。
- 敏感字段仅展示服务端返回值，不尝试还原或缓存完整身份证号、手机号和地址。
- 请求失败、空数据和真实数值 0 分开呈现，不以假数据或 0 掩盖错误。

## 暂不实现

- 微信 OpenID、手机号一键登录、居民注册和身份绑定。
- 人口、家庭户及成员的新增、编辑、删除、离户和户主变更。
- 完整专业申请创建流程及任何专业 `execute` 操作。
- 用户、角色、部门、行政区划和字典管理。
- 大屏完整迁移、批量导出管理、支付、消息订阅、地图、人脸、证照、二维码和政务联网。
- 正式小程序发布审核。

## 验收边界

V1 验收覆盖五个演示账号、登录生命周期、七类页面、权限入口、分页/刷新、审批乐观锁、材料查看、统一错误提示和隐私缓存约束。微信开发者工具与真机测试必须由具备相应环境和网络条件的人员执行，未执行时不得标记为通过。
