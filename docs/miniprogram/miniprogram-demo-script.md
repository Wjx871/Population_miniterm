# 微信小程序 V1 答辩演示脚本

建议时长 8–10 分钟。演示前启动后端、准备一条材料已核验的待审批申请，并清空小程序会话。

## 1. 定位与登录（1 分钟）

说明这是内部移动管理端，不是公众办事小程序，不使用 OpenID。使用 `admin/123456` 登录，展示 login → `/api/auth/me` 两步认证、角色、数据范围和按权限生成的入口。

## 2. 工作台与查询（2 分钟）

展示真实人口、家庭户、待审批数量和工作台常用入口。通过底部“业务”进入业务中心，说明入口按 `population:view`、`household:view` 权限动态生成；再进入人口查询，按姓名查询、下拉刷新、加载下一页并查看脱敏详情和当前家庭关系。返回后进入家庭户查询，展示中文状态、家庭地址和成员关系。

## 3. 我的申请（2 分钟）

通过底部“办理”进入办理中心，展示按权限出现的“我的申请”。进入后展示状态筛选、申请基础信息、专业摘要、材料核验状态、文件预览和审批轨迹。强调专业记录缺失只显示局部提示，不导致整页白屏；待审批字段缺失或接口失败不会被显示成虚假 0。

## 4. 移动审批（3 分钟）

切换为 `approver/123456`，从“办理”进入真实审批列表和待审批详情，核对申请、专业摘要、材料和轨迹。演示驳回原因必填，随后对准备好的记录执行通过或驳回。指出请求携带后端最新 `version`，重复处理返回 409。

审批通过后展示明确提示“业务尚未执行”，并说明审批人员没有迁移、注销、重点人口或敏感导出的专业 execute 权限。

## 5. 权限与退出（1 分钟）

切换为 `viewer/123456`，确认工作台和办理中心不显示审批入口；无办理权限时显示规范空状态。通过底部“我的”展示角色、数据范围、权限摘要和后端健康状态。退出后确认底部导航消失、回到登录页，旧 Token 失效，且只记住用户名、不保存密码。

## 6. 收尾说明（1 分钟）

说明 V1 未实现 OpenID、公众注册、人口/家庭户写操作、复杂申请创建、专业 execute 和正式发布审核；展示开发指南中的真机/合法域名配置及测试报告中仍需人工完成的项目。

演示前还需在微信开发者工具核对：四个 Tab 高亮、iPhone/安卓安全区、二级页不显示底栏、详情返回后的选中状态、退出与重新登录、Network 无重复请求、页面栈无异常增长、控制台无错误。
