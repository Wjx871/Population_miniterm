# 微信小程序 V1 答辩演示脚本

建议时长 8–10 分钟。演示前启动后端、准备一条材料已核验的待审批申请，并清空小程序会话。

## 1. 定位与登录（1 分钟）

说明这是内部移动管理端，不是公众办事小程序，不使用 OpenID。使用 `admin/123456` 登录，展示 login → `/api/auth/me` 两步认证、角色、数据范围和按权限生成的入口。

## 2. 工作台与查询（2 分钟）

展示真实人口、家庭户、待审批数量和工作台常用入口。通过底部“业务”进入业务中心，说明入口按 `population:view`、`household:view` 权限动态生成；再进入人口查询，按姓名查询、下拉刷新、加载下一页并查看脱敏详情和当前家庭关系。返回后进入家庭户查询，展示中文状态、家庭地址和成员关系。

## 3. 我的申请（2 分钟）

通过底部“办理”进入办理中心，展示当前待办和按权限出现的“我的申请”。进入后展示状态筛选、申请卡片的下一步提示，以及详情中的申请摘要、材料、办理进度和审批结果。重点演示审批通过后仍显示“等待业务办理”，只有业务实际完成后才显示“业务已办理完成”；筛选无结果可一键清除条件。

## 4. 移动审批（3 分钟）

切换为 `approver/123456`，从“办理”进入审批列表，展示“待我审批/已处理”、真实待办数量以及“进入审批/查看结果”的动作区分。进入详情后核对申请说明、业务信息、材料和办理记录；演示驳回原因必填、字数提示、通过或驳回前二次确认，以及事项状态变化时自动刷新。

审批通过后展示明确提示“等待业务办理”，说明审批完成不等于业务办结。

## 5. 权限与退出（1 分钟）

切换为 `viewer/123456`，确认工作台和办理中心不显示审批入口；无办理权限时显示规范空状态。通过底部“我的”展示用户、组织信息、中文工作职责、真实可用功能、版本和隐私说明，确认没有英文权限码或运行环境信息。退出后确认底部导航消失、回到登录页，旧 Token 失效，且只记住用户名、不保存密码。

## 6. 收尾说明（1 分钟）

说明 V1 未实现 OpenID、公众注册、人口/家庭户写操作、复杂申请创建、专业 execute 和正式发布审核；展示开发指南中的真机/合法域名配置及测试报告中仍需人工完成的项目。

演示前还需在微信开发者工具核对：四个 Tab 高亮、iPhone/安卓安全区、二级页不显示底栏、详情返回后的选中状态、退出与重新登录、Network 无重复请求、页面栈无异常增长、控制台无错误。
