# 微信小程序移动工作台设计说明

## 产品定位

小程序定位为面向人口管理工作人员的“移动查询 + 轻量申请 + 待办审批 + 进度跟踪”工作台。它复用正式后端的 JWT、权限码、数据范围和审批流程，不复制 Web 管理端的复杂业务能力。

Web 端继续负责人口档案新增与核心字段维护、家庭户创建与成员调整、完整迁入迁出、注销执行、复杂材料核验、专业业务显式执行、批量导入导出、用户角色权限和系统配置。小程序负责高频只读查询、一个轻量签注申请、申请进度、审批决定及账号权限说明。

## 信息架构

底部仍为四个原生自定义 Tab：

1. 工作台：真实指标、更新时间、权限化快捷入口和刷新。
2. 业务：人口、家庭户、居住证、流动人口、迁移记录查询。
3. 办理：我的申请、居住证签注申请、审批办理。
4. 我的：账号、组织、权限摘要、可用功能、系统信息和退出登录。

新增二级页包括居住证列表/详情/签注、流动人口列表/详情、迁移列表/详情和完整权限页。人口详情增加当前居住证及最近三条迁移摘要。

## 轻量申请选择

本轮选择 `RESIDENCE_PERMIT_ENDORSEMENT`（居住证签注）。正式后端已经具备创建草稿、两类必需材料上传、提交、审批、申请查询和 Web 显式执行的完整链路，适合移动端三步办理。小程序不会调用专业执行接口。

流程为：

1. 从真实即将到期证件或指定有效证件中选择办理对象。
2. 填写标题、居住事由、申请原因和可选备注，创建真实草稿。
3. 上传“当前居住证”和“持续居住证明”，提交审批并进入申请详情。

审批人员仍在现有移动审批页执行通过或驳回。审批通过只进入“待业务执行”，最终签注由 Web 端显式完成；小程序随后读取办结结果。

## 状态与错误设计

- 列表区分 loading、empty、error、loading-more 和 end，不把请求失败转换为空列表。
- 缺失指标和总数显示“暂无”或“数量暂不可用”，真实 `0` 保持为 `0`。
- 申请时间线仅按真实申请状态和审批日志生成，不预先展示不存在的节点。
- `401` 清理会话并回登录页；`403` 仅提示无权限；`400/404/409/500` 使用统一安全文案。
- 创建、上传、提交、撤回和审批均有进行中锁；冲突后只由用户操作触发刷新或跳转，不循环重试。

## 权限矩阵

| 移动能力 | 权限码 |
| --- | --- |
| 工作台统计 | `statistics:view` |
| 人口查询 | `population:view` |
| 家庭户查询 | `household:view` |
| 居住证查询 | `residence-permit:view` |
| 居住证到期提醒 | `residence-permit:expiry:view` |
| 流动人口查询 | `floating:view` |
| 迁移记录查询 | `migration:view` |
| 我的申请/进度 | `application:view` |
| 撤回申请 | `application:withdraw` |
| 发起居住证签注 | `residence-permit:apply` |
| 审批查看 | `approval:view` |
| 审批处理 | `approval:handle` |

入口和按钮只依赖后端返回的权限码，不根据中文角色名放宽能力。数据范围继续由后端执行。

## 隐私与本地状态

服务端脱敏值原样展示；对于意外返回的未脱敏证件号、身份证号和手机号，移动适配器再做防御性掩码。人口及家庭户敏感业务数据不写入本地持久缓存。退出登录沿用正式接口并清理 Token 和用户会话。页面不输出 Token、完整身份证号或固定业务记录 ID。

## 未搬入小程序的能力

未实现复杂迁入迁出录入、注销、家庭户调整、人口核心档案编辑、专业业务执行、材料核验、批量导出、用户权限配置和系统配置。这些能力继续留在 Web 端，以避免移动端绕过业务校验或扩大数据权限。

## 后续建议

- 后端若增加正式消息中心，可将实时业务提醒升级为可追溯消息列表。
- 若需要更精确的签注资格提示，可增加只读资格检查接口；在此之前仍以后端创建时的正式校验为准。
- 微信开发者工具服务端口开启后，补齐多角色真机/预览截图和自动化编译记录。
