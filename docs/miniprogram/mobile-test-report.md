## 错误与边界

- `401`：清理会话且并发场景只重定向一次。
- `403`：保留会话并显示无权限提示。
- `400/404/409/500`：统一为安全、可理解的用户提示。
- 请求失败：保留错误态和用户触发的重试入口，不显示空数据或假零。
- 重复操作：首次加载、刷新、分页、创建、上传、提交、撤回和审批均有保护。
- 申请审批通过：显示等待 Web 专业执行，不显示虚假办结。

## 微信开发者工具与人工验收

迁移基线验证时已定位到微信开发者工具 CLI，但 IDE 服务端口未开启，因此 CLI 编译和自动化预览被环境阻塞，未宣称通过。完成代码后仍需在开发者工具中人工覆盖 admin、approver 和只读/普通业务角色：登录、四个 Tab、各查询页、签注上传与提交、审批通过/驳回、申请办结回看、Token 失效、无权限、底部安全区及不同屏宽。

测试不得使用真实敏感截图或在代码中写入账号密码。当前仓库未提交测试截图、视频、AppSecret、Token、私有项目配置或本地接口配置。

## 已知限制

1. 微信开发者工具 CLI 编译、预览和多角色人工结果仍需在 IDE 开启服务端口后补测。
2. 小程序不执行签注、迁移、注销等专业业务；这是明确产品边界，不是缺陷。
3. 后端当前没有独立消息中心，本轮未虚构消息记录。
