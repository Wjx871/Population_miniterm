# 微信小程序 UI 与一级导航设计

## 信息架构

一级导航固定为四项：

1. 工作台：用户摘要、六项真实指标和四个常用快捷入口。
2. 业务：人口与家庭户基础档案入口。
3. 办理：我的申请、审批办理以及真实可用的待审批摘要。
4. 我的：用户与组织信息、工作职责、可用功能、版本说明和退出登录。

人口、家庭户、申请、审批的列表和详情继续作为二级页面，不占用底部导航。登录页也不属于 Tab 页面。

## 导航实现

- 使用微信原生 `custom-tab-bar` 配置，不在普通页面里实现伪底栏。
- 一级页切换使用 `wx.switchTab`，二级页使用 `wx.navigateTo`，退出登录使用 `wx.reLaunch`。
- 四个一级页在 `onShow` 通过公共 `utils/tab-bar.js` 同步选中项，退出时重置为工作台。
- 公共 `.tab-page` 统一为内容预留底栏及 `safe-area-inset-bottom` 高度。

## 入口与权限

| 页面 | 入口 | 权限码 | 现有路由 |
| --- | --- | --- | --- |
| 业务 | 人口信息 | `population:view` | `/pages/persons/list/index` |
| 业务 | 家庭户 | `household:view` | `/pages/households/list/index` |
| 办理 | 我的申请 | `application:view` | `/pages/applications/list/index` |
| 办理 | 审批办理 | `approval:view` | `/pages/approvals/list/index` |

入口仅由权限码过滤，不根据 `ADMIN`、`VIEWER`、`APPROVER` 或中文角色名判断。办理 Tab 对所有已登录用户固定存在；没有办理权限时显示公共空状态。

迁入、迁出、居住证、流动人口、重点人口和独立统计当前没有对应的完整小程序页面与路由，因此不展示，也不创建空壳或“敬请期待”入口。

## 视觉规范

底栏使用白色背景、浅边框和轻阴影；选中项为 `#1677FF`，未选中项为 `#8A94A6`。四个图标来自本地 Tabler Outline 资源，统一为 40rpx，不使用字符图标、远程字体或网络资源。

业务与办理页面采用适合少量真实入口的高质量纵向卡片，而非填充空位的宫格。待审批摘要只在现有接口返回有限数值时显示：真实 0 显示为 0，字段缺失不虚构 0，接口失败使用公共错误状态并由用户触发重试。

## 验证状态

自动测试和静态检查覆盖页面数量、四个 Tab、颜色、图标、安全区、权限过滤、精确路由、空状态、真实 0/缺失值、请求失败与首次请求去重。

微信开发者工具和真机仍需人工确认：Tab 实际显示与点击、高亮状态、安全区、内容遮挡、不同账号权限差异、二级页底栏隐藏、详情返回高亮、退出与重新登录、控制台和网络请求、横向溢出、图标裁切及页面栈增长。

第六阶段命令行复核时，开发者工具 CLI 因服务端口未开启而无法连接，上述项目仍保持“待人工验收”，不记为已通过。截图尺寸、安全区和长文本效果应以实际模拟器/真机为准。

## 用户任务导向规范

- 页面按用户任务组织，不按服务路径或数据结构组织；首屏优先回答“当前状态是什么、下一步做什么”。
- 普通用户界面不展示数据库、缓存、服务实现、网络状态码、英文权限码或内部字段名；这些信息只保留在开发日志、测试与技术文档中。
- “我的申请”使用原始申请状态判断审批与业务办理进度。审批通过只表示审核完成，只有原始状态为 `COMPLETED` 才显示业务已办结。
- “审批办理”区分待我审批与已处理，待办卡使用“进入审批”，历史卡使用“查看结果”；审批意见驳回必填，提交前二次确认。
- “个人中心”把权限码转换为中文工作职责和真实可用功能，不展示运行环境或基础设施状态。
- 空状态同时说明原因和可执行下一步；加载失败提供重新加载，筛选无结果提供清除条件。
