## 1. 项目概述

本版是“人口数据库管理系统”内部工作人员的微信小程序移动管理端。V1 复用现有 JWT、角色权限、数据范围和后端接口，提供移动查询、申请进度查看和审批处理。它不是公众办事小程序，不使用 OpenID，不扩大现有后端业务范围。用户可见版本为 `V1.0.0`。

## 3. 导航结构

四个一级 Tab 为“工作台、业务、办理、我的”，使用原生 `custom-tab-bar` 和 `wx.switchTab`。人口、家庭户、申请和审批的列表/详情为二级页，通过 `wx.navigateTo` 进入。退出和 401 使用 `wx.reLaunch` 回到登录页。

## 5. 核心业务流程

- 登录：账号密码 → login 取得会话 → `/auth/me` 恢复真实身份和权限。
- 档案查询：业务中心 → 人口/家庭户列表 → 详情 → 关联档案。
- 申请跟踪：办理中心 → 我的申请 → 材料、进度、审批意见和下一步。
- 审批：办理中心 → 待我审批 → 核对信息/材料 → 确认通过或驳回。审批通过不等于业务已办结。
- 退出：用户确认 → 后端 logout 尽力执行 → 本地会话清理 → 重定向登录页。

## 6. 用户视角设计原则

页面先说明当前状态和下一步，不暴露权限码、网络状态码、数据库、缓存或服务实现。真实 0 与数据不可用分开；业务判断只使用原始状态；空状态和错误状态都提供用户可执行的下一步。图标使用已审核的 Tabler Icons 官方资源，本地打包，不使用字符、Emoji、远程字体或手绘资源。

## 14. 已知限制

- V1 不包含 OpenID、公众注册、人口/家庭户写操作、复杂申请创建、专业业务 execute 和尚未移动化的迁移/居住证/重点人口页面。
- 真机不能使用电脑的 `127.0.0.1`；正式发布需可信 HTTPS 域名及微信合法域名配置。
- 微信开发者工具双机型、Console/Network、13 页截图和真机未完成。
- Vite 大 chunk 和依赖注释警告属既有 PC 前端构建警告，本阶段未扩大范围重构。

## 15. 发布结论

代码和自动化门禁达到当前 V1 的可用与答辩准备标准；功能分支可推送并用于创建草稿 PR。但在微信开发者工具双机型验收、Console/Network 清零和 13 页真实截图完成前，不建议标记为“最终人工验收通过”或直接合并 `develop`。合并前还必须处理 `origin/develop` 的独有提交并重跑全量回归。
