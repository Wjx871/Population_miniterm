# 人口数据库管理系统前端统一开发约束

> 文档版本：V1.0  
> 适用范围：人口数据库管理系统 Web 管理端  
> 技术栈：Vue 3 + Vite + Element Plus  
> 文档用途：统一前端页面范围、权限规则、交互规范、组件标准和开发边界，作为后续开发、联调和验收依据。

---

## 1. 文档目标

本项目不是简单的“每张数据表做一个 CRUD 页面”，而是围绕以下业务主线组织前端：

1. 人口档案管理；
2. 家庭户口及户籍关系管理；
3. 迁入、迁出和注销业务办理；
4. 流动人口、居住凭证、证件和重点人口管理；
5. 重大业务申请、审批和结果落表；
6. 综合查询、统计展示和数据导出；
7. 用户、权限、数据字典和日志管理。

前端开发必须优先保证：

- 页面名称、字段名称、状态名称统一；
- 菜单权限、按钮权限、接口权限保持一致；
- 重大操作必须进入申请审批流程；
- 当前户籍与历史归档严格区分；
- 身份证号、手机号等敏感信息默认脱敏；
- 同类页面使用统一布局、组件和交互方式。

## 2. 三层权限模型

本系统当前只设计三种权限等级，不再单独设置“审批人员”角色。审批能力并入最高权限。

| 权限等级 | 统一角色名称 | 角色代码建议 | 权限定位 |
|---|---|---|---|
| 一级 | 普通用户 | `NORMAL_USER` | 以查询和查看为主 |
| 二级 | 户口管理员 | `HOUSEHOLD_ADMIN` | 办理日常人口、户籍及扩展业务 |
| 三级 | 超级管理员 | `SUPER_ADMIN` | 拥有全部功能、全部数据和审批权限 |

角色代码需与后端最终定义保持一致，前端不得自行维护另一套角色名称。

### 2.1 一级权限：普通用户

普通用户主要用于查询、统计和信息查看。

允许：

- 登录系统；
- 查看个人权限范围内的工作台；
- 查询人口、家庭户、迁移、证件等非敏感信息；
- 查看允许访问的数据详情；
- 查看基础统计结果；
- 修改个人密码和个人联系方式；
- 进行普通、非敏感数据导出（后端允许时）。

禁止：

- 新增、修改、删除人口和户籍数据；
- 办理迁入、迁出、销户、注销等业务；
- 登记重点人口；
- 审批业务申请；
- 查看完整身份证号、完整手机号等敏感字段；
- 管理用户、角色、权限、部门、字典和日志；
- 发起或下载敏感数据导出。

### 2.2 二级权限：户口管理员

户口管理员可以完成日常业务管理，但重大操作必须提交给三级权限审批。

允许直接办理的常规业务：

- 新增、修改人口基础信息；
- 家庭户立户和基础信息维护；
- 家庭成员添加、移出和关系维护；
- 户主设置和普通变更；
- 当前户籍信息查询；
- 流动人口登记和居住信息维护；
- 居住凭证、通用证件管理；
- 普通迁入迁出申请的填写、保存和提交；
- 重点人口业务申请的填写和提交；
- 查看本人提交的申请和审批进度；
- 普通条件查询、分页和授权范围内的数据导出。

禁止：

- 直接执行重大操作；
- 审批本人或他人的业务申请；
- 直接销户、人口注销或删除有效户籍；
- 直接恢复已归档、已注销的数据；
- 查看或导出超出数据范围的敏感信息；
- 管理系统用户、角色、权限、部门和全局配置。

### 2.3 三级权限：超级管理员

超级管理员拥有系统最高权限。

允许：

- 访问全部业务和系统管理页面；
- 办理所有日常业务；
- 审批、退回、驳回重大业务申请；
- 执行销户、人口注销、敏感导出等重大操作；
- 查看完整敏感信息；
- 管理用户、角色、部门、权限和数据字典；
- 查看全部操作日志、审批日志和导出日志；
- 访问全部数据范围；
- 对异常业务进行恢复、纠正和人工处理。

即使超级管理员可以直接办理重大业务，也必须生成完整的审批或审计记录，不允许绕过日志。

## 3. 权限实现统一约束

### 3.1 权限判断来源

前端权限必须以后端登录接口返回的数据为准，至少包括：

```json
{
  "token": "xxx",
  "user": {
    "userId": 1,
    "username": "admin",
    "realName": "超级管理员",
    "roleCode": "SUPER_ADMIN",
    "permissionLevel": 3,
    "departmentId": 1
  },
  "permissions": [
    "person:query",
    "person:add",
    "household:update",
    "approval:approve"
  ]
}
```

前端不得通过用户名、页面路径或写死的账号判断权限。

### 3.2 权限控制层级

前端至少实现四层控制：

1. **路由权限**：无权访问的路由不得注册或必须跳转 403；
2. **菜单权限**：无权限菜单不得显示；
3. **按钮权限**：无权限操作按钮不得显示；
4. **字段权限**：敏感字段根据权限脱敏或隐藏。

前端隐藏按钮只负责交互控制，后端必须再次校验接口权限。

### 3.3 路由元信息规范

所有业务路由统一使用以下元信息：

```ts
meta: {
  title: '人口档案',
  icon: 'User',
  requiresAuth: true,
  permission: 'person:query',
  minLevel: 1,
  keepAlive: true,
  hidden: false
}
```

### 3.4 按钮权限规范

统一使用权限指令或权限函数：

```vue
<el-button v-permission="'person:add'">新增人口</el-button>
```

禁止在页面中大量出现以下写法：

```vue
v-if="user.role === 'admin'"
```

角色只表示权限集合，具体按钮应尽量根据权限码控制。

### 3.5 数据范围

数据范围由后端控制，前端不得自行扩大。

前端需要做到：

- 不缓存未授权的完整数据；
- 不通过修改查询参数访问其他部门数据；
- 下拉选择器只展示后端返回的可选范围；
- 超级管理员可查看全量数据；
- 普通用户和户口管理员只能查看后端授权范围内的数据。

## 4. 重大操作审批规则

### 4.1 默认纳入审批的重大操作

以下操作必须由二级权限发起申请，由三级权限审批后执行：

- 家庭户销户；
- 人口注销；
- 有效户籍注销或归档；
- 跨区迁入、跨区迁出或迁往市外；
- 重点人口登记或解除；
- 三级敏感数据导出；
- 批量删除、批量归档或批量状态变更；
- 恢复已注销、已归档的重要业务数据；
- 其他由后端标记为重大业务的操作。

普通人口修改、家庭成员维护、流动人口登记、证件维护等常规操作不进入重大审批流程。

### 4.2 审批状态

统一使用以下状态：

| 状态代码 | 状态名称 | 可执行操作 |
|---|---|---|
| `DRAFT` | 草稿 | 编辑、删除、提交 |
| `SUBMITTED` | 已提交 | 查看、允许时撤回 |
| `UNDER_REVIEW` | 审批中 | 查看、允许时撤回 |
| `RETURNED` | 已退回 | 修改、补充材料、重新提交 |
| `REJECTED` | 已驳回 | 查看审批意见、重新发起 |
| `APPROVED` | 已通过 | 等待业务执行 |
| `COMPLETED` | 已办结 | 查看业务结果 |
| `WITHDRAWN` | 已撤回 | 编辑或重新提交 |

页面不得自行创建其他同义状态。

### 4.3 标准审批流程

```text
户口管理员填写业务
        ↓
保存草稿
        ↓
上传和校验材料
        ↓
提交申请
        ↓
业务数据冻结，不允许直接修改
        ↓
超级管理员审批
   ┌────┼────┐
   ↓    ↓    ↓
通过  退回  驳回
   ↓    ↓
执行  补充后重新提交
业务
   ↓
生成业务记录、归档记录和操作日志
```

### 4.4 页面交互要求

- 二级权限点击重大操作时，不得直接弹出“确认执行”；
- 必须进入申请表单或申请步骤条；
- 提交前展示操作影响，例如“审批通过后当前户籍将归档”；
- 提交后业务表单进入只读状态；
- 审批通过后，由后端事务完成状态联动；
- 前端不得先删除当前数据再等待审批；
- 退回和驳回时审批意见必填；
- 所有审批结果使用时间线展示。

## 5. 总体菜单结构

左侧菜单必须使用分组折叠结构，不允许所有菜单平铺。

```text
工作台
├─ 工作台
└─ 数据大屏

人口户籍
├─ 人口档案
├─ 家庭户口
├─ 当前户籍
└─ 历史归档

业务办理
├─ 迁入管理
├─ 迁出管理
├─ 注销管理
└─ 我的申请

审批中心
└─ 审批工作台

扩展业务
├─ 流动人口
├─ 居住凭证
├─ 重点人口
└─ 证件管理

查询统计
└─ 综合查询

系统管理
├─ 用户管理
├─ 角色与权限
├─ 部门管理
```

## 7. 公共页面

### 7.1 登录页

路由：`/login`

功能：

- 用户名、密码登录；
- 记住账号；
- 表单校验；
- 登录失败提示；
- 账号禁用提示；
- 登录成功后保存 Token、用户信息和权限；
- 根据权限跳转工作台；
- 已登录用户访问登录页时自动跳转首页。

### 7.2 403 无权限页

路由：`/403`

功能：

- 提示无权访问；
- 返回工作台；
- 返回上一页；
- 不展示敏感路由信息。

### 7.3 404 页面

路由：`/:pathMatch(.*)*`

功能：

- 提示页面不存在；
- 返回工作台；
- 返回上一页。

## 10. 业务办理模块

### 10.1 迁入管理

路由：`/business/migration-in`

标签：

- 迁入记录；
- 新建迁入申请。

新建流程：

```text
1. 选择迁入类型
2. 选择或登记人员
3. 填写来源信息
4. 选择目标家庭户
5. 填写迁入原因
6. 上传申请材料
7. 确认并提交
```

校验：

- 目标家庭户必须有效；
- 人员不能已有当前有效户籍；
- 跨区迁入必须符合区划规则；
- 必需材料未上传时不得提交；
- 提交后不得直接修改。

### 10.2 迁出管理

路由：`/business/migration-out`

流程：

```text
1. 选择当前户籍人员
2. 展示原户籍信息
3. 选择迁出类型
4. 填写迁往地址
5. 填写迁出原因
6. 上传材料
7. 提交审批
```

校验：

- 户主迁出必须先处理户主变更；
- 办结后生成户籍历史快照；
- 当前户籍由后端事务移除；
- 不允许前端先删后审。

### 10.3 注销管理

路由：`/business/cancellation`

标签：

- 人口注销；
- 家庭户销户。

人口注销：

- 选择人员；
- 查看当前户籍；
- 选择注销原因；
- 填写注销日期；
- 上传材料；
- 提交审批；
- 查看归档结果。

家庭户销户：

- 选择家庭户；
- 自动检查当前成员；
- 自动检查有效户籍；
- 上传材料；
- 提交审批；
- 查看销户结果。

注销管理仅对二级和三级权限开放。

### 10.4 我的申请

路由：`/business/my-application`

标签：

- 全部；
- 草稿；
- 审批中；
- 已退回；
- 已通过；
- 已办结。

功能：

- 新建业务申请；
- 保存草稿；
- 继续编辑；
- 删除草稿；
- 上传和删除材料；
- 提交申请；
- 撤回申请；
- 重新提交；
- 查看审批进度；
- 查看办结结果。

申请详情：

```text
申请基本信息
目标人员或家庭户
```

## 21. 接口调用统一规范

### 21.1 请求封装

统一通过 `src/utils/request.ts` 发送请求。

必须实现：

- Token 自动携带；
- 统一超时；
- 401 自动退出登录；
- 403 跳转无权限页；
- 统一错误消息；
- 防止重复提示；
- 文件下载处理；
- 请求取消或页面卸载处理。

### 21.2 响应格式

建议统一：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

分页格式建议：

```json
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "records": [],
    "total": 0,
    "pageNum": 1,
    "pageSize": 10
  }
}
```

前端不得为不同页面编写不同的分页解析逻辑。

### 21.3 错误处理

- 表单错误显示在对应字段；
- 普通业务错误使用消息提示；
- 删除、注销、审批等关键操作使用确认框；
- 系统错误显示统一错误页或通知；
- 不向用户直接展示后端堆栈信息；
- 错误提示必须说明下一步怎么处理。

## 28. 最终统一原则

1. 三层权限固定为普通用户、户口管理员、超级管理员；
2. 超级管理员拥有全部权限，并承担重大业务审批；
3. 户口管理员只能直接办理常规业务；
