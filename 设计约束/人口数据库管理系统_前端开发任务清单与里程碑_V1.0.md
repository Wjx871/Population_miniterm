# 人口数据库管理系统前端开发任务清单与里程碑 V1.0

> 项目仓库：`Wjx871/Population_miniterm`  
> 前端分支：`feat/frontend-population`  
> 适用范围：仅包含张锦源负责的前端设计、页面实现、查询统计、扩展业务前端及前后端联调  
> 技术栈：Vue 3 + Vite + Element Plus + Pinia + Vue Router + Axios + ECharts  
> 执行原则：先完善前端骨架与权限展示，再打通核心业务页面，最后补查询统计、大屏和演示优化。

---

## 1. 前端工作边界

### 1.1 本人负责

- Vue 3 前端工程维护；
- 登录页和后台整体布局；
- 菜单、路由、面包屑；
- 登录状态保存；
- 三级权限对应的菜单和按钮显示；
- 人口、家庭户口、迁入迁出、审批中心等核心页面；
- 流动人口、居住凭证、证件、重点人口页面；
- 综合查询、分页、筛选、导出入口；
- 首页工作台和 ECharts 数据大屏；
- 表单校验、错误提示、空状态、权限不足页面；
- 接口字段对齐和前后端联调；
- 演示数据和答辩流程整理。

### 1.2 不属于本人主责

以下内容由后端或数据库负责人完成，前端只负责调用接口和展示结果：

- Spring Boot 后端架构；
- 登录鉴权算法；
- 接口权限校验；
- 审批状态流转；
- 迁入迁出事务；
- 销户和注销事务；
- 当前户籍转历史归档；
- 数据库建表、约束和索引；
- 操作日志和导出日志写入；
- 数据范围最终校验；
- 文件存储和下载鉴权。

## 3. 前端主链路目标

最小可用前端应稳定完成：

```text
登录
  -> 根据三级权限显示菜单
  -> 人口档案查询与维护
  -> 家庭户和成员维护
  -> 迁入/迁出申请填写
  -> 查看我的申请
  -> 超级管理员进入审批中心
  -> 查看审批结果
  -> 查询当前户籍和历史记录
```

扩展展示链路：

```text
流动人口登记
  -> 居住凭证/证件管理
  -> 重点人口业务
  -> 综合查询
  -> 首页工作台
  -> 数据大屏
```

# 6. M1 后台框架与三级权限展示

## 目标

完成统一后台框架，并按三级权限控制菜单、路由和按钮显示。

三级权限：

| 等级 | 角色 | 前端表现 |
|---|---|---|
| 一级 | 普通用户 | 以查询和查看为主 |
| 二级 | 户口管理员 | 可办理常规管理业务，可提交重大业务申请 |
| 三级 | 超级管理员 | 可查看全部菜单并进入审批和系统管理 |

## 任务清单

| ID | 任务 | 优先级 | 产出 |
|---|---|---|---|
| M1-01 | 扩展用户 Store | P0-core | 保存角色、权限等级、权限码 |
| M1-02 | 实现登录状态恢复 | P0-core | 页面刷新后保持登录 |
| M1-03 | 实现路由守卫 | P0-core | 未登录跳登录，无权限跳 403 |
| M1-04 | 实现三级菜单过滤 | P0-core | 不同角色显示不同菜单 |
| M1-05 | 实现按钮权限指令 | P0-core | 新增、编辑、审批按钮按权限显示 |
| M1-06 | 重构侧边栏为分组菜单 | P0-demo | 菜单层级清晰 |
| M1-07 | 增加面包屑 | P0-demo | 页面位置清晰 |
| M1-08 | 增加 403 和 404 页面 | P0-core | 权限和错误路由有明确反馈 |
| M1-09 | 顶部用户区改为下拉菜单 | P1 | 个人中心、修改密码、退出 |

## 建议菜单

```text
工作台

人口户籍
├─ 人口档案
├─ 家庭户口
├─ 当前户籍
└─ 历史归档

业务办理
├─ 迁入管理
├─ 迁出管理
├─ 注销管理
└─ 我的申请

审批中心
└─ 审批工作台

扩展业务
├─ 流动人口
├─ 居住凭证
├─ 重点人口
└─ 证件管理

查询统计
└─ 综合查询

系统管理
├─ 用户管理
├─ 数据字典
└─ 其他管理页面
```

## 验收标准

1. 普通用户看不到新增、编辑、审批和系统管理。
2. 户口管理员能进入业务办理，但看不到审批中心。
3. 超级管理员可以看到全部菜单。
4. 手动输入无权限路由时进入 403。
5. 未登录访问业务页时跳转登录页。

| M3-06 | 实现申请详情 | P0-core | 表单、材料、审批时间线 |
| M3-07 | 实现撤回和重新提交入口 | P1 | 申请状态操作 |

| M3-12 | 根据审批结果刷新业务页面 | P0-core | 办结结果可查看 |

## 标准流程

```text
户口管理员填写申请
  -> 保存草稿
  -> 上传材料
  -> 提交
  -> 超级管理员审批
  -> 通过 / 退回 / 驳回
  -> 查看最终结果
```

## 验收标准

1. 户口管理员可以提交迁入或迁出申请。
2. 提交后页面进入只读状态。
3. 超级管理员可以进入审批中心。
4. 退回时可以查看意见并重新提交。
5. 审批通过后可以查看最终业务结果。
6. 前端不直接删除迁移记录或当前户籍。
